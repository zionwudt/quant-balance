# 知衡（QuantBalance）Web 端设计规范

> 面向个人量化投资者的专业级 Web 体验。

---

## 设计理念

### 一句话定位

**克制的专业感** —— 不是花哨的营销页，而是一个让人信赖的交易研究工具。

### 核心原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **数据密度优先** | 屏幕上每一像素都应传递信息，减少装饰留白 | 巨大的 hero banner、纯装饰插画 |
| **操作零猜测** | 用户看到任何控件都能立刻理解它做什么 | 含义模糊的图标、隐藏的右键菜单 |
| **结果即反馈** | 操作后 <300ms 给出视觉反馈，长任务显示进度 | 点击后无响应、突然跳转 |
| **错误可恢复** | 所有报错附带解决建议，输入出错不清空已填内容 | "发生未知错误"、表单清空 |

### 参考基调

取以下产品的交集：
- **TradingView** 的图表交互密度
- **Linear** 的极简操作流
- **Bloomberg Terminal** 的信息架构（但用现代视觉语言）
- **Notion** 的排版节奏感

## 视觉系统

### 配色

采用**深色主题为主、浅色主题可切换**的双模式。深色是量化/交易类工具的用户预期。

#### 深色模式（默认）

```
背景层级:
  --bg-base:      #0a0a0f       ← 最底层，接近纯黑带微蓝
  --bg-surface:   #12121a       ← 卡片/面板底色
  --bg-elevated:  #1a1a28       ← 弹窗/下拉/hover 底色
  --bg-input:     #0f0f18       ← 输入框底色

文字:
  --text-primary:   #e8e8ed     ← 主文字，不用纯白避免刺眼
  --text-secondary: #8888a0     ← 次要说明
  --text-muted:     #555568     ← 禁用/占位符

边框与分割:
  --border:         #1e1e2e     ← 卡片边框
  --border-subtle:  #16161f     ← 微弱分割线

语义色:
  --profit:     #22c55e         ← 盈利/涨（绿色，国际惯例）
  --loss:       #ef4444         ← 亏损/跌（红色）
  --accent:     #6366f1         ← 主操作按钮（靛蓝，中性专业）
  --accent-hover: #818cf8
  --warning:    #f59e0b         ← 警告/注意
  --info:       #3b82f6         ← 信息提示
```

> **涨跌色说明**：A 股用户习惯红涨绿跌，但技术圈/量化圈多数用国际惯例。
> 提供设置项让用户自选：`涨跌色风格: 国际(绿涨红跌) / A股(红涨绿跌)`。

#### 浅色模式

```
  --bg-base:        #f8f9fc
  --bg-surface:     #ffffff
  --bg-elevated:    #f0f1f5
  --text-primary:   #1a1a2e
  --text-secondary: #6b7280
  --border:         #e5e7eb
  其他语义色保持不变
```

### 字体

```css
/* 数据/数字 —— 等宽字体确保对齐 */
--font-mono: "JetBrains Mono", "SF Mono", "Cascadia Code", monospace;

/* 界面文字 —— 系统字体栈，中文优先 */
--font-sans: -apple-system, BlinkMacSystemFont,
             "PingFang SC", "Noto Sans SC", "Microsoft YaHei",
             sans-serif;

/* 字号体系 */
--text-xs:    0.75rem;    /* 12px - 标签、辅助文字 */
--text-sm:    0.8125rem;  /* 13px - 表格内容、次要信息 */
--text-base:  0.875rem;   /* 14px - 正文（比常规网页小一号，信息密度更高）*/
--text-lg:    1rem;       /* 16px - 小标题 */
--text-xl:    1.25rem;    /* 20px - 区块标题 */
--text-2xl:   1.5rem;     /* 24px - 页面标题 */
--text-3xl:   2rem;       /* 32px - 仅用于核心数字展示 */
```

### 间距 & 圆角

```css
/* 间距基于 4px 网格 */
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-6: 24px;
--space-8: 32px;

/* 圆角 —— 偏小，保持专业锐利感 */
--radius-sm: 4px;     /* 按钮、输入框 */
--radius-md: 6px;     /* 卡片 */
--radius-lg: 8px;     /* 弹窗 */

/* 投影 —— 深色模式下用微弱亮边代替阴影 */
--shadow-card: 0 0 0 1px var(--border);
--shadow-elevated: 0 4px 24px rgba(0, 0, 0, 0.4);
```

### 动效

```css
/* 全局过渡基准 */
--transition-fast: 120ms ease-out;   /* hover、toggle */
--transition-base: 200ms ease-out;   /* 面板展开、tab 切换 */
--transition-slow: 350ms ease-out;   /* 页面切换、图表加载 */

/* 原则：
   - 不做纯装饰动画（无 bounce、无 spring）
   - 数据加载中用 skeleton shimmer，不用 spinner
   - 图表数据更新用 morph 动画，避免重绘闪烁
*/
```

#### 交互细节

**股票代码输入框**：
- 输入时弹出下拉搜索（支持代码 + 名称模糊搜索）
- 如 `"茅台"` → 显示 `600519.SH 贵州茅台`
- 选中后显示为 Tag 样式：`[600519.SH 贵州茅台 ×]`
- 组合模式下支持多选（逗号分隔或多次选择）

**日期选择**：
- 日历面板选择，非交易日灰色不可选
- 快捷按钮：近 1 年 / 近 3 年 / 近 5 年 / 今年以来 / 全部
- 选择范围后实时显示交易日天数：`共 245 个交易日`

**策略切换**：
- 选择策略后，参数区**平滑过渡**为对应参数表单
- 每个策略附带一行灰字说明，如 `"DIF 上穿 DEA 时买入，下穿时卖出"`

**运行回测按钮**：
- 默认态：实心靛蓝按钮 `▶ 运行回测`
- 运行中：按钮变为进度条样式 `⏳ 回测中...`，参数区不可编辑
- 完成：按钮短暂变绿 `✓ 完成`，然后恢复，结果区滚动到视图

**高级设置（折叠区）**：

```
── 高级设置 ──
  滑点模式    [无 ▾]  [百分比 ▾]  滑点率 [0.1] %
  止损比例    [  8  ] %    ☐ 启用
  止盈比例    [ 20  ] %    ☐ 启用
  单票最大仓位 [ 20  ] %
  最大持仓数   [  5  ] 只
  基准指数    [000300.SH 沪深300 ▾]
```

#### 指标卡片区

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│ 总收益    │ │ 年化收益  │ │ 最大回撤  │ │ Sharpe   │ │ 胜率     │ │ 交易次数  │
│ +23.5%   │ │ +18.2%   │ │ -12.4%   │ │  1.38    │ │ 62.5%   │ │   24     │
│ 绿色     │ │ 绿色     │ │ 红色     │ │ 绿色     │ │ 绿色     │ │ 中性     │
└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘

规则：
- 数字用 --font-mono 等宽字体，32px 大号
- 正收益 / 高 Sharpe / 高胜率 → --profit 色
- 负收益 / 高回撤 → --loss 色
- 中性指标 → --text-primary
- hover 显示 tooltip 解释含义
```

#### 权益曲线图

**必备交互**：
- 双曲线：策略净值 + 基准净值（基准为半透明虚线）
- 回撤区域：净值与峰值之间的区域用淡红色填充
- 鼠标 hover：十字光标 + tooltip 显示日期、净值、回撤%、基准值
- 缩放：鼠标滚轮缩放时间轴，拖拽平移
- 区间选择：框选一段时间，显示该段的收益统计

**图表库**：ECharts（生态成熟、中文文档好、支持 Canvas/SVG 双渲染）

#### K 线图

**必备功能**：
- 标准 OHLC K 线 + 成交量柱状图
- 买入点：绿色上箭头标记 ▲
- 卖出点：红色下箭头标记 ▼
- 均线叠加：策略使用的均线自动叠加显示
- 鼠标十字光标 + 数据面板（日期、开高低收、成交量）
- 缩放与拖拽

**图表库**：ECharts K 线组件 或 Lightweight Charts（更轻量、交互更流畅）

#### 月度收益热力图

```
         1月   2月   3月   4月   5月   6月   7月   ...  12月
  2023  [+2.1] [-0.8] [+3.4] [+1.2] [-2.1] [+0.5] ...
  2024  [+1.5] [+4.2] [-1.1] [+2.8] [+0.3] [-3.2] ...

颜色映射：
  深绿 ← +5% 以上
  浅绿 ← +1% ~ +5%
  灰白 ← -1% ~ +1%
  浅红 ← -5% ~ -1%
  深红 ← -5% 以下
```

#### 成交明细表

```
┌────┬────────────┬──────┬────────┬────────┬─────────┬─────────┬──────────┐
│ #  │ 股票       │ 方向  │ 入场日期 │ 出场日期 │ 入场价   │ 出场价   │ 盈亏      │

│ 2  │ 600519.SH  │ 买入  │ 02-03  │ 02-20  │ 1710.00 │ 1685.20 │ -1.45%   │

└────┴────────────┴──────┴────────┴────────┴─────────┴─────────┴──────────┘

交互：
- 固定表头，内容滚动
- 点击某行 → K线图上高亮对应买卖区间
- 盈利行左侧有绿色竖线，亏损行红色竖线
```

## 技术实现方案

### 架构选型

```
方案: 单 HTML 入口 + ES Modules（无构建工具）

理由:
  - 项目是 Python 后端为主，前端不应引入 Node/npm 构建链
  - 减少部署复杂度（pip install 后直接可用）
  - 现代浏览器原生支持 ES Modules，性能足够
```

### CDN 依赖（同时提供本地 fallback）

| 库 | 版本 | 用途 | 大小 |
|----|------|------|------|
| ECharts | 5.5+ | 权益曲线、热力图、饼图 | ~800KB (gzip ~280KB) |
| Lightweight Charts | 4.x | K 线图（可选，ECharts 也可） | ~45KB |

不引入任何 UI 框架（React/Vue），用原生 DOM + ES Modules，减少包体积和复杂度。

### FastAPI 静态文件服务

```python
# app.py 新增
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse

app.mount("/static", StaticFiles(directory="web/static"), name="static")

@app.get("/")
def dashboard():
    return FileResponse("web/index.html")
```

### 前后端通信约定

```
基础规则:
  - 所有 API 返回 JSON，前端 fetch + JSON.parse
  - 成功: HTTP 200，body 为数据
  - 客户端错误: HTTP 400，body {"detail": "错误描述（中文）"}
  - 服务端错误: HTTP 500，body {"detail": "内部错误，请查看日志"}
  - 长任务: POST 返回 {"task_id": "xxx"}，前端轮询 GET /api/tasks/{id}

Loading 状态管理:
  - 请求发出 → 对应区域显示 skeleton
  - 响应到达 → skeleton 渐隐，数据渐入 (transition 200ms)
  - 超时 (30s) → 显示重试按钮
```
